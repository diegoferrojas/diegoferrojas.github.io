Artifact: Implementation Model
Purpose
The implementation model is a composite, comprehensive artifact which encompasses all artifacts needed to build and manage the system in the run-time environment.

Description
Main Description The  implementation model  is a collection of components, and the implementation subsystems that contain them. Components include both deliverable components, such as executables, and components from which the deliverables are produced, such as source code files.
